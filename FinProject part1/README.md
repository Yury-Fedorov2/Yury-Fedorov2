# Приложение интерактивного анализа метеорологических данных

##  Описание приложения

Streamlit-приложение для анализа и визуализации исторических метеоданных.  
Позволяет исследовать температуру, осадки, ветер и дождливые дни с помощью интерактивных графиков, фильтров и вычисляемых показателей.

##  Функционал

| Функция | Реализация |
|---------|-------------|
| **Календарь для выбора диапазона дат** | Виджет `st.sidebar.date_input` – выбор начальной и конечной даты |
| **Таблица с фильтрацией** | `st.dataframe` отображает отфильтрованные данные (по городам и датам) с возможностью сортировки и постраничного просмотра |
| **Производные столбцы** | Добавлены: категория температуры (холодно/умеренно/жарко), уровень осадков, бинарный признак комфортности погоды |
| **EDA – распределения признаков** | Гистограммы (температура, осадки, ветер) + boxplot температуры |
| **Сравнение городов** | Столбчатая диаграмма средних значений температуры, осадков и ветра (при выборе нескольких городов) |
| **Временные ряды и прогноз** | График динамики выбранного показателя + **скользящее среднее** (окно настраивается от 3 до 30 дней), которое служит простым прогнозом тренда |
| **Интерактивность** | Мультивыбор городов, выбор показателя, слайдер окна скользящего среднего, переключатель агрегации (день/неделя/месяц) |

##  Как пользоваться приложением

1. Активируйте зависимости:
   ```bash
   .venv\Scripts\activate
   ```
   
2. Запустите приложение из папки проекта:
   ```bash
   streamlit run FP1_weather_analyzer.py
   ```
3. Используйте боковую панель для настройки:

Выберите один или несколько городов.

Установите диапазон дат (календарь).

Выберите показатель для временного ряда (температура / осадки / ветер).

Задайте окно скользящего среднего (чем больше окно – тем сильнее сглаживание).

4. Основные вкладки и блоки:

Таблица – просмотр отфильтрованных данных с производными колонками.

EDA – гистограммы и boxplot (вкладка "Распределение признаков") + сравнение городов (вкладка "Сравнение городов").

Динамика и прогноз – график фактических значений и скользящего среднего.

Интерактивные метрики – сводная статистика и график агрегированных данных по выбранному периоду.

Все графики и таблицы обновляются автоматически при изменении фильтров.

 Структура базы данных
Приложение ожидает файл data/weather.db с таблицей weather, содержащей следующие колонки:

date (TEXT или DATE)

city (TEXT)

avg_temp (REAL) – средняя температура

total_precip (REAL) – общее количество осадков

avg_wind (REAL) – средняя скорость ветра

is_rainy (INTEGER) – 1 если дождь, иначе 0
